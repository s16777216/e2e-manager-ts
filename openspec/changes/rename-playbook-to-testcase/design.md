## Context

系統前端將測試項目的中文呈現稱為「劇本」，但在代碼、變數、API 與 Database 實體中，該概念均以 `Testcase` 來命名。這種前後端名稱的不一致在維護時容易引起困惑，且在軟體測試業界中，「測試案例 (Test Case)」更具專業度與通用性。本變更著眼於前端 UI 的中文名詞替換與語意統一。

## Goals / Non-Goals

**Goals:**
- **中文 UI 替換**：在前端所有可見的 UI 視圖、標題、選單、引導說明與對話方塊中，將「劇本」替換為「測試案例」或其簡稱「案例」。
- **提示與吐司通知 (Toast) 修正**：將 Hook 和請求完成提示中的「劇本」字眼改為「測試案例」（例如「劇本修改成功」改為「測試案例修改成功」）。
- **保留 IDE cross-header 對齊**：維持前一變更所完成的 Bento IDE 網格對齊佈局。

**Non-Goals:**
- **代碼變數與型別不變**：不修改任何後端 API 欄位、Database schema、資料模型實體與程式變數命名（如 `Testcase`、`testcaseCount`、`api.createTestcase` 等維持不變），以降低重構風險。
- **無功能性邏輯變更**：不涉及任何功能邏輯的改動。

## Decisions

### 1. 前端中文靜態替換
- **決策**：針對全域 `grep_search` 出來的前端程式碼中的中文字詞「劇本」進行靜態替換。
- **原因**：中文名詞修改不影響任何業務邏輯，僅需修改呈現層。不更動程式內部變數命名可確保 100% 的 backward compatibility。

### 2. 局部文案簡化（視排版寬度）
- **決策**：在空間有限的 UI（如表格欄位「劇本數」），改為「案例數」，以維持 Bento layout 空間的和諧度。

## Risks / Trade-offs

- **[Risk]** 部分動態文案或漏掉的部分仍顯示舊名詞。
  - **[Mitigation]** 透過 grep 進行全面靜態代碼掃描，確保所有含有「劇本」二字的程式行皆被列出並修復，並以瀏覽器 subagent 進行完整畫面掃描。
